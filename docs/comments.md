# Как работают комментарии

## Самый простой бесплатный режим

1. Читатель пишет комментарий в форме под статьей.
2. Браузер открывает письмо на адрес из `PELICAN_COMMENT_SYSTEM_EMAIL_USER` и `PELICAN_COMMENT_SYSTEM_EMAIL_DOMAIN`.
3. Ты проверяешь письмо вручную. Это и есть модерация и защита от спама.
4. Если комментарий нормальный, создаешь файл:

```text
content/comments/slug-stati/002-korotkoe-imya.md
```

5. Формат файла:

```markdown
email: reader@example.com
date: 2026-07-22T18:10:00+03:00
author: Имя читателя
website: https://example.com

Текст комментария.
```

Для ответа на другой комментарий добавь поле `replyto`. Значение - имя файла родительского комментария без точки перед расширением. Например для `001-test.md`:

```markdown
replyto: 001-testmd
```

После `git push` GitHub Actions пересоберет сайт, и комментарий появится на странице.

## Почему не Staticman по умолчанию

Staticman умеет принимать HTML-форму и отправлять pull request в репозиторий, но ему нужен живой Node.js-сервер. Старые инструкции предлагают Heroku, но бесплатный Heroku больше не является надежной базой для полностью бесплатного блога. Поэтому в проекте включен email-режим: он проще, бесплатный, с ручной модерацией и без внешней зависимости.

## Если все-таки нужен Staticman

Можно поднять свой Staticman instance на бесплатном тарифе Render/Fly.io или другом хостинге, затем заменить `mailto`-форму в `themes/blog/templates/comments.html` на `POST` в Staticman API. В этом режиме Staticman будет создавать pull request с новым файлом комментария, а ты будешь принимать или закрывать pull request.
